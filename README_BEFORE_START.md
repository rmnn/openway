Поскольку в условии задачи ничего не сказано про то, какие будут началные данные, я решил сделать их в удобном для себя формате:
это матрица NxM размера. 
Формат входящих данных:
два числа heigh и width - высота и ширина лабиринта (heigh * width < 64 * 64).
Далее следует матрица height*width содержашая нули и еденицы.
0 - на эту клетку можно вступать, 1 - нельзя.
После матрицы следуют координаты начальной точки и конечной.
Формат выходящий : число, равное длине минимального пути в лабиринте.
